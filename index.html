Overview

The Bridge Inspection Ontology (BrIO) provides a formal vocabulary for bridge inspection planning and bridge inspection data management in digital twins for bridge maintenance. BrIO extends OCQA with bridge-specific concepts such as bridge component inspections, inspection types, inspection characteristics, inspection procedures, inspection equipment, regulations, damages, and inspection records.

The ontology represents inspections from a bridge component perspective, where each component can be assessed with respect to condition, environmental exposure, quantity, and inspector recommendations. It also supports the description of inspection types, including inventory, routine, in-depth, damage, and special inspections, as defined by relevant regulations.

In addition, BrIO includes an example case for a bridge inspection scenario together with RDF instance data and SPARQL queries. The example resources are available here: Example case and SPARQL queries.
